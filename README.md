# 家庭车库智能充电机器人 - 路径跟踪控制仿真

## 项目简介

本项目实现了一个**全向轮移动机器人**在车库环境中的**路径跟踪控制仿真系统**。该系统模拟机器人从起点自主导航到充电桩位置的完整过程，适用于：

- 🏆 大学生机器人竞赛
- 📚 控制算法课程设计  
- 🔬 移动机器人导航研究
- 💡 自动充电系统开发

### 核心特性

✅ **模块化设计** - 清晰的代码结构，易于理解和扩展  
✅ **完整的性能评估** - 10+ 项控制性能指标  
✅ **参数自动优化** - 自动化参数对比实验  
✅ **任务状态机** - 完整的充电任务流程管理  
✅ **高质量可视化** - 适合论文和PPT展示的图表

## 技术方案

### 机器人平台

**底盘类型**：三轮全向轮（正三角形布局）

```
        轮1 (0°)
         ↑
        / \
       /   \
      /  ●  \    ● = 机器人中心
     /       \
    ↙         ↘
 轮2 (120°)  轮3 (240°)
```

**运动学特点**：
- 可实现任意方向的平移运动
- 无需转向即可横向移动
- 适合狭窄空间和精确对接

**运动学方程**：

```
[ω1]   [  sin(0°)    -cos(0°)    -R ] [vx]
[ω2] = [  sin(120°)  -cos(120°)  -R ] [vy]
[ω3]   [  sin(240°)  -cos(240°)  -R ] [ω ]
```

其中：
- `ω1, ω2, ω3` - 三个车轮的角速度
- `vx, vy` - 机器人在自身坐标系下的速度
- `ω` - 机器人的角速度
- `R` - 机器人中心到车轮的距离

### 控制算法

**控制器类型**：位置-航向解耦控制

**控制策略**：
1. **位置控制**：根据机器人到参考路径的距离，生成速度指令
2. **航向控制**：根据期望朝向与当前朝向的偏差，生成角速度指令
3. **前视控制**：在路径上选取前视点，引导机器人跟踪

**控制方程**：

```python
# 位置误差（机器人坐标系）
error_x = (target_x - robot_x) * cos(yaw) + (target_y - robot_y) * sin(yaw)
error_y = -(target_x - robot_x) * sin(yaw) + (target_y - robot_y) * cos(yaw)

# 速度控制
vx = position_gain * error_x
vy = position_gain * error_y

# 航向控制
desired_yaw = atan2(target_y - robot_y, target_x - robot_x)
omega = heading_gain * (desired_yaw - robot_yaw)
```

### 性能指标

| 指标类别 | 指标名称 | 单位 | 物理意义 |
|---------|---------|------|---------|
| **位置跟踪** | 平均位置误差 | m | 机器人实际轨迹到参考路径的平均偏离距离 |
| | 最大位置误差 | m | 跟踪过程中出现的最大偏离距离 |
| | 最终到位误差 | m | 机器人最终位置与目标点的距离 |
| | 稳态误差 | m | 轨迹后20%阶段的平均误差 |
| **航向跟踪** | 平均航向误差 | ° | 机器人朝向与期望朝向的平均偏差 |
| | 最大航向误差 | ° | 航向偏差的最大值 |
| **路径效率** | 路径效率 | % | 参考路径长度/实际路径长度×100% |
| **时间性能** | 总仿真时间 | s | 从起点到终点的总用时 |
| | 稳定时间 | s | 误差首次进入并保持在阈值内的时间 |

## 参数调优指南

### 位置控制增益 (position_gain)

**作用**：控制对位置误差的响应速度

**推荐范围**：1.0 - 2.5

**调优建议**：
- **增大** → 快速纠正偏差，但可能产生振荡
- **减小** → 轨迹更平滑，但响应较慢
- **默认值**：1.5（平衡性能）
- **激进值**：2.5（最佳跟踪精度）

### 航向控制增益 (heading_gain)

**作用**：控制对航向误差的响应速度

**推荐范围**：1.5 - 3.0

**调优建议**：
- **增大** → 快速对齐路径方向
- **减小** → 转向更平滑
- **默认值**：2.0（平衡性能）
- **激进值**：3.0（快速响应）

### 前视距离 (look_ahead_distance)

**作用**：控制跟踪点的前瞻距离

**推荐范围**：0.5 - 1.2 m

**调优建议**：
- **增大** → 轨迹更平滑，但可能切弯
- **减小** → 更紧密跟随路径
- **默认值**：0.8 m（平衡性能）
- **精确值**：0.5 m（最佳跟踪精度）

### 参数组合推荐

根据参数对比实验结果，推荐以下参数组合：

| 应用场景 | position_gain | heading_gain | look_ahead | 特点 |
|---------|--------------|--------------|------------|------|
| **平衡型**（默认） | 1.5 | 2.0 | 0.8 | 综合性能良好 |
| **激进型**（推荐） | 2.5 | 3.0 | 0.5 | 最佳跟踪精度 |
| **保守型** | 1.0 | 1.5 | 1.2 | 平滑但响应慢 |
| **精确型** | 2.0 | 2.5 | 0.6 | 优化定位精度 |
| **平滑型** | 1.2 | 2.5 | 1.0 | 轨迹平滑 |

## 状态机设计

### 状态定义

系统包含 5 个状态，模拟完整的充电任务流程：

| 状态 | 说明 | 进入条件 | 退出条件 | 速度 |
|------|------|----------|----------|------|
| **IDLE** | 空闲等待 | 初始状态 | 任务启动 | 0% |
| **MOVE_TO_TARGET** | 路径跟踪移动 | 任务启动 | 距离目标 < 0.5m | 100% |
| **ALIGN_TO_DOCK** | 精确对准 | 接近目标 | 位置误差 < 0.05m 且航向误差 < 5° | 30% |
| **DOCKED** | 对接完成 | 对准成功 | 停留 2.0 秒 | 0% |
| **FINISHED** | 任务完成 | 充电完成 | 无（终止状态） | 0% |

### 状态转换流程

```
    IDLE
      ↓ (任务启动)
MOVE_TO_TARGET
      ↓ (距离 < 0.5m)
ALIGN_TO_DOCK
      ↓ (精确对准)
    DOCKED
      ↓ (停留 2s)
   FINISHED
```

## 常见问题

### Q1: 如何修改路径类型？

编辑 `src/config.py` 文件：

```python
path_type: str = "L_shape"  # 可选: "straight", "circle", "L_shape"
```

### Q2: 如何调整仿真时间？

编辑 `src/config.py` 文件：

```python
total_time: float = 30.0  # 仿真总时长（秒）
dt: float = 0.05          # 仿真步长（秒）
```

### Q3: 如何获得更高分辨率的图表？

在 `run_demo.py` 中修改 DPI 参数：

```python
dpi=300  # 默认300，可改为600获得更高分辨率
```

### Q4: 仿真不收敛怎么办？

尝试以下方法：
- 降低 `position_gain` 和 `heading_gain`
- 增大 `look_ahead_distance`
- 检查路径是否有效
- 运行参数对比实验找到最优参数

### Q5: 如何添加自定义路径？

在 `src/path_planner.py` 中添加新方法：

```python
@staticmethod
def _generate_custom_path() -> np.ndarray:
    """生成自定义路径"""
    # 定义路径点
    x = np.array([0, 1, 2, 3, 4, 5])
    y = np.array([0, 0, 1, 2, 2, 2])
    return np.column_stack([x, y])
```

然后在 `generate_path` 方法中注册：

```python
elif path_type == "custom":
    path = PathPlanner._generate_custom_path()
```
